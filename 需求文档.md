# 中医处方结算系统 - 需求文档

## 1. 项目概述

### 1.1 项目背景
开发一套适用于中医诊所的处方开具与结算管理系统，支持移动端优先的操作体验，可在安卓手机端部署，通过局域网提供服务。

### 1.2 项目目标
- 实现处方的快速开具与结算
- 支持药材库存管理与预警
- 提供数据统计与分析功能
- 支持离线使用，数据本地存储
- 可通过手机浏览器或局域网电脑浏览器访问

### 1.3 技术选型
| 项目 | 技术 |
|------|------|
| 前端框架 | Vue 3 + Vite |
| 状态管理 | Pinia |
| 本地存储 | IndexedDB (LocalForage) |
| 路由 | Vue Router |
| 日期处理 | Day.js |
| 打印导出 | html2canvas + jsPDF |
| PWA | vite-plugin-pwa |

### 2.3 处方开具模块

#### 2.3.1 处方信息
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String | 自动 | 唯一标识符 |
| patientId | String | 否 | 关联患者ID |
| patientName | String | 是 | 患者姓名 |
| diagnosis | String | 是 | 病症诊断 |
| advice | String | 否 | 用药建议/医嘱 |
| dosage | Number | 是 | 剂数，默认1 |
| usage | String | 否 | 服用方法 |
| items | Array | 是 | 药材列表，含herbId/herbName/dosage/originalPrice/finalPrice |
| consultFee | Number | 否 | 诊费 |
| originalTotal | Number | 自动 | 药材原价合计（未折扣） |
| finalTotal | Number | 自动 | 结算金额（含诊费） |
| costTotal | Number | 自动 | 成本合计 |
| profit | Number | 自动 | 利润 |
| paymentMethod | String | 是 | 支付方式：cash/wechat/alipay/other |
| paidAmount | Number | 是 | 实收金额 |
| status | String | 自动 | 状态：completed |
| createdAt | Date | 自动 | 开方时间 |
| updatedAt | Date | 自动 | 更新时间 |

#### 2.3.2 药材明细(items)字段
| 字段 | 类型 | 说明 |
|------|------|------|
| herbId | String | 药材ID |
| herbName | String | 药材名称 |
| dosage | Number | 剂量（克） |
| originalPrice | Number | 单剂原价 |
| finalPrice | Number | 单剂结算价 |

#### 2.3.3 价格计算规则
```
药材单价 = 销售价 / 1000 (元/克)
单剂原价 = 药材单价 × 剂量
单剂结算价 = 原价 × (折扣 / 10)
药材小计 = Σ(各药材单剂结算价)
处方结算金额 = 药材小计 × 剂数 + 诊费
```

#### 2.3.4 开方方式
1. **手动开方**
   - 搜索选择药材
   - 输入剂量
   - 自动计算价格

2. **模板开方**
   - 提供3种内置模板：
     - 感冒发热方
     - 调理脾胃方
     - 安神助眠方
   - 选择模板后可修改

3. **图片识别（OCR）**
   - 拍照或上传处方图片
   - 自动识别药材名称和剂量
   - 人工确认后使用
   - **注意：需要联网，离线时提示用户**

## 3. 非功能需求

### 3.1 性能要求
- 页面加载时间 < 3秒
- 药材/患者列表支持虚拟滚动（数据量大时）
- 图片压缩后上传（OCR功能）

### 3.2 兼容性
- Chrome / Edge（推荐）
- Safari（iOS）
- 安卓 WebView
- 支持触摸操作

### 3.3 安全性
- 数据本地存储，不上传服务器
- 敏感操作需确认（删除、清空）
- 定期提醒用户备份数据

### 3.4 UI/UX
- 移动端优先设计
- 底部 Tab 导航
- 简洁现代风格
- 主色调：青绿色（#10B981）

## 7. 项目构建与打包

### 7.1 环境要求

| 工具 | 版本要求 | 说明 |
|------|----------|------|
| Node.js | >= 18.0.0 | JavaScript 运行时 |
| npm | >= 9.0.0 | Node.js 包管理器 |
| Android Studio | 2022+ | Android 开发工具（打包APK用） |
| Java JDK | 17+ | Java 开发环境 |

### 7.2 本地开发运行

```bash
# 1. 进入项目目录
cd tcm-prescription-system

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev

# 4. 访问项目
# 本机: http://localhost:3000
# 局域网: http://<本机IP>:3000
```

### 7.3 构建 Web 应用

```bash
# 构建生产版本（输出到 dist 目录）
npm run build
```

### 7.4 打包 Android APK

#### 7.4.1 初始化 Capacitor（仅首次需要）

```bash
# 安装 Capacitor CLI
npm install -g @capacitor/cli

# 初始化 Capacitor（如果 android 目录已存在可跳过）
npx cap init
```

#### 7.4.2 构建流程

```bash
# 步骤1: 构建 Web 应用
npm run build

# 步骤2: 同步 Web 资源到 Android 项目
npx cap sync android

# 步骤3: 编译 APK
cd android
# Windows:
gradlew.bat assembleDebug
# Mac/Linux:
./gradlew assembleDebug

# 步骤4: 获取 APK
# 位置: android/app/build/outputs/apk/debug/tcmps.apk
```

#### 7.4.3 一键打包脚本

```bash
# 在项目根目录执行（Windows PowerShell）
npm run build && npx cap sync android && cd android; .\gradlew.bat assembleDebug
```

### 7.5 APK 安装与测试

1. **传输 APK 到手机**
   - 通过数据线复制
   - 发送到微信文件传输助手
   - 通过 QQ/钉钉等应用

2. **安装 APK**
   - 首次安装需要允许"安装未知来源应用"
   - 鸿蒙系统：设置 → 安全 → 允许安装未知来源应用

3. **测试检查项**
   - 页面加载正常
   - 数据存储正常（IndexedDB）
   - 文件读写功能（需要存储权限）
   - 打印功能

### 7.6 常见问题

| 问题 | 解决方案 |
|------|----------|
| gradlew 命令找不到 | 使用 `gradlew.bat`（Windows）或 `./gradlew`（Mac/Linux） |
| Android SDK 未配置 | 设置 ANDROID_HOME 环境变量 |
| APK 安装失败 | 检查手机是否已开启"允许安装未知来源应用" |
| 文件写入失败 | 检查应用存储权限，鸿蒙系统需额外授权 |
